# 2026-05-22 工作记录

## Color Game 前端实现

为用户刘工实现了一个完整的 Color Game（颜色骰子）网页游戏：

### 第一版
- PixiJS + HTML + CSS 实现
- 单颜色下注
- 静态骰子展示

### 第二版（重构）
- 支持多颜色下注：每次点击颜色按钮增加1个筹码（紫色圆形筹码堆叠在按钮上）
- 3D CSS骰子：使用 `transform-style: preserve-3d` 实现真正的3D立方体
- 骰子下落动画：纯JS requestAnimationFrame控制，从斜坡顶部滚落到底部，带弹跳效果
- 3个骰子颜色面排列各不相同，增加随机感
- 右下角清除按钮（重置所有下注）
- 底部金额/余额显示栏
- 完整游戏逻辑：下注→扣款→摇骰→按匹配数派彩（x2/x3/x16.7）

### 第三版（图片版）
- 使用 `pc_dark_bg.png` 作为全屏背景（扑克桌面暗色风格）
- 使用 `box.png` (314x499) 替换 CSS 画的假机器，直接用美术图
- 骰子定位根据 box.png 内部空间重新校准：从箱子内部斜坡顶部滚落到底部
- 派彩面板位置微调以适应新布局

### 技术要点
- CSS 3D transform 实现立方体骰子（6个面，translateZ + rotateX/Y）
- JS动画代替CSS keyframes，更精确控制下落轨迹和旋转
- 每个骰子独立的面颜色映射，确保结果正确显示
- 筹码堆叠用绝对定位+margin-top偏移实现
- 图片资源使用相对路径引用，单HTML+图片即可运行

### 第四版（PixiJS重构）
- 脚本代码提取到 `game.js`，HTML 只保留结构和样式
- 移除耳机按钮及所有音频相关代码
- 3D骰子改用 PixiJS WebGL 渲染（等距投影立方体）
  - `IsoDice` 类：每个骰子独立 PIXI.Application（透明背景）
  - 立方体8顶点 + 6面，旋转矩阵实时计算，Painter's Algorithm 排序面
  - 动画用 `requestAnimationFrame` 驱动，三颗骰子错开250ms启动
  - 下落轨迹 ease-in，旋转逐渐减速，落地弹跳效果
- 已知问题：`game.js` 动画逻辑较复杂，如需调试请检查浏览器控制台
- 文件清单：`color-game.html` + `game.js` + `box.png` + `pc_dark_bg.png`

### 第五版（CSS 3D 骰子重写）
- 用户反馈5个问题：骰子太大、面透明、停止时面斜着、重开闪白、重开后无动画
- 完全移除 PixiJS，改用 CSS 3D DOM 骰子（`transform-style: preserve-3d`）
- 解决面透明问题：`backface-visibility: hidden` 正确剔除背面
- 解决停止时面朝上问题：用 `FACE_TARGET_ANGLES` 表精确控制每个面停止角度
- 解决闪白问题：DOM 元素无 canvas 重建闪白
- 解决重开后无动画问题：正确重置 `isRolling` 状态，动画逻辑用 `requestAnimationFrame` 驱动
- 骰子尺寸 `DICE_SIZE = 21`（原42的一半）
- 随机概率控制模块：
  - `setDiceProbability({ mode:'weighted', weights:[2,1,1,1,1,1] })` 加权随机
  - `setDiceProbability({ mode:'force', results:[0,0,0] })` 强制结果（测试用）
  - `setDiceProbability({ mode:'default' })` 恢复均匀随机
  - 代码位置已用注释 `【BEGIN】随机概率控制` / `【END】随机概率控制` 标记
- `pixi.min.js` 已下载到本地（454KB，v7.3.2），但当前版本已不使用
- 文件清单：`color-game.html` + `game.js` + `box.png` + `pc_dark_bg.png`
